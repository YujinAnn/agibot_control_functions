# mujoco_x2_cpp

Native C++ MuJoCo sim of the Agibot X2 (31 free joints), exposing the same ROS HAL
interface as `mujoco_x2.py`. Renders on the GPU so it hits real-time (RTF ~1.0).

ROS interface:
- subscribes `/aima/hal/joint/{head,waist,arm,leg}/command` (`aimdk_msgs/JointCommandArray`)
- publishes  `/aima/hal/joint/{head,waist,arm,leg}/state`   (`aimdk_msgs/JointStateArray`)
- publishes  `/aima/hal/imu/{chest,torso}/state`            (`sensor_msgs/Imu`)

outputs regenerated by `colcon build` (see below).

## Prerequisites (installed on the system, not shipped in this repo)

- ROS 2 Humble
- `aimdk_msgs`  вЂ” custom message package; build/source it separately
- MuJoCo        вЂ” the pip/conda `mujoco` package (provides `libmujoco.so` + headers)
- GLFW, yaml-cpp, OpenGL

## Build

```bash
source /opt/ros/humble/setup.bash
source /path/to/aimdk/install/setup.bash          # provides aimdk_msgs
export MUJOCO_DIR=/path/to/site-packages/mujoco   # or just activate the conda env with mujoco

cd <repo-root>/mujoco_x2
colcon build --paths .                            # creates build/ install/ log/ here
```

`MUJOCO_DIR` is resolved in this order: `-DMUJOCO_DIR=...`, then the `MUJOCO_DIR` env
var, then the `mujoco/` dir of the active conda env (`$CONDA_PREFIX`). If none is
found the configure step fails with a clear message.

## Run

```bash
source install/setup.bash
run_mujoco_x2              # GLFW viewer
run_mujoco_x2 --no-viewer # headless
run_mujoco_x2 --print     # log pelvis-z / RTF at ~1 Hz
```

## Notes on relocatability

The binary locates its data files at runtime вЂ” it searches upward from the executable's
directory (and the current directory) for `robots/x2/robot_full_flat_ground_excl.xml`.
So the repo can be renamed or moved anywhere and run from any working directory. To point
it at a specific root explicitly, set the `MX2_REPO_ROOT` env var.

If you move or rename the repo, delete `mujoco_x2/{build,install,log}` and re-run
`colcon build --paths .` вЂ” the old build artifacts bake in absolute paths and won't
follow the move. The source and data never need editing.
